## ü§ù Contributing

### Adding New Components

1. Create the component in the appropriate directory
2. Add TypeScript types and props documentation
3. Create a `.stories.tsx` file with comprehensive examples
4. Include usage guidelines and accessibility notes
5. Test all variants and states
6. Review accessibility with the a11y addon

### Updating Existing Components

1. Update the component implementation
2. Update the story file with new props/variants
3. Update documentation and examples
4. Test all existing stories still work
5. Review accessibility impact

### Documentation

- Write clear component descriptions
- Provide usage guidelines
- Include code examples
- Document accessibility features
- Show common patterns and anti-patterns

## üí° Best Practices
